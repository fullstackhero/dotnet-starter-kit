Get the starter kit running locally in under 5 minutes.

<Aside type="tip" title="Prerequisites">
Make sure you have [.NET 10 SDK](https://dot.net/download) and [Docker](https://www.docker.com/) installed before you begin. See [Prerequisites](/dotnet-starter-kit/prerequisites/) for details.
</Aside>

## Installation <Badge>Pick one</Badge>

<Tabs labels={["FSH CLI (Recommended)", "dotnet new Template", "Clone Repository"]}>
<TabPanel>

Install the FSH CLI tool and create a new project with the interactive wizard:

```bash
dotnet tool install -g FullStackHero.CLI
fsh doctor    # verify your environment
fsh new MyApp
cd MyApp
```

The wizard lets you choose your database provider (PostgreSQL/SQL Server) and whether to include the Aspire AppHost.

Other useful commands:

```bash
fsh info      # show versions and available updates
fsh update    # update CLI and template to latest
```

</TabPanel>
<TabPanel>

Install the template and scaffold a new project:

```bash
dotnet new install FullStackHero.NET.StarterKit
dotnet new fsh -n MyApp
cd MyApp
```

Available template options:

```bash
dotnet new fsh -n MyApp --db sqlserver    # use SQL Server instead of PostgreSQL
dotnet new fsh -n MyApp --aspire false    # skip Aspire AppHost
```

</TabPanel>
<TabPanel>

Clone the repository directly:

```bash
git clone https://github.com/fullstackhero/dotnet-starter-kit.git MyApp
cd MyApp
```

<Aside type="note">
When cloning directly, project names retain the `FSH.Starter` prefix. The CLI and template options rename everything to your chosen project name.
</Aside>

</TabPanel>
</Tabs>

## Run the Application <Badge>2 min</Badge>

<Steps>

### Start the Application

<Tabs labels={["With Aspire (Recommended)", "Standard", "Docker"]}>
<TabPanel>

```bash
dotnet run --project src/Host/MyApp.AppHost
```

Aspire orchestrates PostgreSQL, Redis, and the API automatically - no manual setup needed. Once it starts, open the Aspire dashboard at the URL shown in your terminal output to monitor all running services, view logs, and inspect traces.

<Aside type="note">
If you cloned the repo directly (instead of using the CLI or template), use `FSH.Starter.AppHost` as the project name.
</Aside>

</TabPanel>
<TabPanel>

```bash
dotnet run --project src/Host/MyApp.Api
```

<Aside type="warning" title="Local Services Required">
This approach requires PostgreSQL and Redis running locally on their default ports. Configure connection strings in `appsettings.json` or via environment variables if your setup differs.
</Aside>

</TabPanel>
<TabPanel>

```bash
docker-compose up -d
```

This builds and runs the entire stack including PostgreSQL, Redis, and the API in containers. No local dependencies needed beyond Docker itself.

</TabPanel>
</Tabs>

### Explore the API

Once the application is running:

- **API base URL** - `https://localhost:7030` (or the port shown in your terminal)
- **Scalar API docs** - `https://localhost:7030/scalar/v1`
- **Health check** - `https://localhost:7030/health`

Open the Scalar UI in your browser to browse all available endpoints, inspect request/response schemas, and execute API calls directly from the documentation interface. It is the fastest way to explore what the starter kit offers out of the box.

</Steps>

## Your First API Call

Start by generating an authentication token against the root tenant:

```bash
curl -X POST https://localhost:7030/api/v1/identity/tokens \
  -H "Content-Type: application/json" \
  -H "X-Tenant-ID: root" \
  -d '{"email": "admin@root.com", "password": "123Pa$$word!"}'
```

Copy the `token` value from the response and use it to call a protected endpoint:

```bash
curl https://localhost:7030/api/v1/identity/users \
  -H "Authorization: Bearer {token}" \
  -H "X-Tenant-ID: root"
```

<Aside type="tip" title="Default Credentials">
The default root tenant comes pre-seeded with an admin user. Use the credentials above to get started.
</Aside>

## Run Tests

```bash
dotnet test src/MyApp.slnx
```

This runs the full test suite including architecture tests (module boundary enforcement via NetArchTest), unit tests, and module-level integration tests.

<Aside type="note">
If you cloned the repo directly, the solution file is `FSH.Starter.slnx`.
</Aside>

<Aside type="warning" title="BuildingBlocks">
Do not modify files under `src/BuildingBlocks/` without understanding the impact - these are shared framework libraries consumed by all modules. Changes there have a wide blast radius across the entire solution.
</Aside>
